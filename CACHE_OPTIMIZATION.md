# 帖子缓存优化方案

## 概述

本次优化实现了完整的二级缓存策略，提升了帖子相关功能的性能。

## 优化内容

### 1. 二级缓存架构

- **本地缓存 (Caffeine)**: 提供最快的访问速度，适合高频访问的数据
- **分布式缓存 (Redis)**: 提供跨实例的数据共享，适合分布式部署

### 2. 缓存策略

#### 帖子详情缓存
- 缓存键: `post:detail:{postId}`
- 本地缓存: 1000条，30分钟过期
- Redis缓存: 2小时过期
- 适用场景: 帖子详情页面访问

#### 帖子列表缓存
- 缓存键: `post:list:{userId}:{orderMode}:{offset}:{limit}`
- 本地缓存: 500条，10分钟过期
- Redis缓存: 30分钟过期
- 适用场景: 首页帖子列表、用户帖子列表

#### 帖子总数缓存
- 缓存键: `post:count:{userId}`
- 本地缓存: 100条，5分钟过期
- Redis缓存: 1小时过期
- 适用场景: 分页计算

### 3. 缓存失效策略

#### 自动失效
- 添加帖子时: 清除相关列表和总数缓存
- 更新帖子时: 清除帖子详情和相关列表缓存
- 更新评论数时: 清除帖子详情和列表缓存
- 更新分数时: 清除帖子详情和热门列表缓存

#### 手动失效
- 通过Actuator端点手动清除指定缓存
- 支持清除帖子详情、列表、总数缓存

### 4. 新增功能

#### 缓存服务类
- `DiscussPostCacheService`: 统一的缓存管理服务
- 实现二级缓存逻辑
- 提供缓存清除方法

#### 缓存配置类
- `CacheConfig`: 统一的缓存配置管理
- 支持通过配置文件调整缓存参数

#### 缓存监控端点
- `CacheEndPoint`: 提供缓存状态查看和手动清除功能
- 访问地址: `/community/actuator/cache`

#### 缓存预热任务
- `CacheWarmUpJob`: 定时预热热门和最新帖子缓存
- 每30分钟执行一次
- 提升系统启动后的响应速度

### 5. 配置文件更新

```properties
# 帖子缓存配置
cache.post.detail.expire-seconds=7200
cache.post.list.expire-seconds=1800
cache.post.count.expire-seconds=3600
cache.local.post.detail.max-size=1000
cache.local.post.list.max-size=500
cache.local.post.count.max-size=100
```

## 性能提升

### 预期效果
- **响应时间**: 缓存命中时响应时间减少80%以上
- **数据库压力**: 减少70%以上的数据库查询
- **并发能力**: 提升3-5倍的并发处理能力


## 使用说明

### 1. 启动应用
应用启动后会自动初始化缓存服务，并开始定时预热任务。

### 2. 监控缓存
访问 `/community/actuator/cache` 查看缓存状态。

### 3. 手动清除缓存
通过Actuator端点手动清除指定缓存：
- 清除帖子详情: POST `/community/actuator/cache?postId=1`
- 清除帖子列表: POST `/community/actuator/cache?userId=0&orderMode=1`
- 清除帖子总数: POST `/community/actuator/cache?userId=0`

## 注意事项

1. **内存使用**: 本地缓存会占用一定内存，请根据服务器配置调整缓存大小
2. **数据一致性**: 缓存失效策略确保数据最终一致性，但可能存在短暂的数据不一致
3. **Redis依赖**: 确保Redis服务正常运行，否则会降级为仅使用本地缓存
4. **监控告警**: 建议配置缓存命中率监控，及时发现缓存问题

## 后续优化建议

1. **缓存穿透防护**: 对空结果进行缓存，避免频繁查询不存在的数据
2. **缓存雪崩防护**: 为缓存过期时间添加随机偏移
3. **缓存预热优化**: 根据访问模式动态调整预热策略
4. **缓存统计**: 添加详细的缓存命中率统计
5. **分布式锁**: 在缓存更新时使用分布式锁避免并发问题
